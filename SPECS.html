🚨 Blocked — Credentials Needed From You

These features are coded and deployed but will not work until you supply the following environment variables in the Vercel dashboard and in .env.local for local development.

BLOCKER 1 — SIGN IN
AUTH_RESEND_KEY + EMAIL_FROM
Magic link emails can't be sent without a Resend API key and a verified sender domain. Get the key at resend.com/api-keys. The EMAIL_FROM must use a domain you own and have verified in Resend (e.g. noreply@yourdomain.com).

Alternative: We can swap to Google OAuth instead (see Open Decisions below) — simpler for a friend group, no domain needed.
BLOCKER 2 — COACH ZARA AI
ANTHROPIC_API_KEY
Coach Zara calls Claude Haiku (claude-haiku-4-5-20251001) via the Anthropic API. Without this key the /coach page will return a 500 error. Get it at console.anthropic.com/api-keys.
BLOCKER 3 — PHOTO UPLOADS
BLOB_READ_WRITE_TOKEN
Profile photos and progress photos upload to Vercel Blob. In Vercel dashboard → Storage → Create a Blob store → Link to this project. The token is auto-added to your Vercel env vars, then copy it to .env.local for local dev.
BLOCKER 4 — GOOGLE OAUTH (if switching from magic links)
GOOGLE_CLIENT_ID + GOOGLE_CLIENT_SECRET
Only needed if we switch from Resend magic links to Google Sign-In (recommended — see Open Decisions). Create an OAuth 2.0 Client ID at console.cloud.google.com → APIs & Services → Credentials → Web application. Add redirect URI: https://120-day-challenge.vercel.app/api/auth/callback/google

🤔 Open Decisions — Need Your Input

DECISION 1 — AUTH STRATEGY

Magic Links (Resend) vs. Google OAuth

Current: Resend magic link emails. Requires: a verified custom domain + Resend API key. Elegant, passwordless.
Alternative: Google Sign-In. Requires: a Google Cloud project (free). Simpler setup — everyone has a Google account. The v2 design prototype uses Google OAuth.
Recommendation: Switch to Google OAuth. It's what the design uses, no domain needed, and easier for the squad.

DECISION 2 — CHALLENGE START DATE

When does the 120 days begin?

The app currently computes "Day N of 120" from the challenge's startDate field. We need you to confirm: what date did the challenge start? (or when should it start?) This determines which day number is shown in the top bar and progress calculations.

DECISION 3 — SQUAD INVITE FLOW

How do new members join?

Built: challenge creation with an invite link. You create a challenge → share the invite URL → friends click it → they join. Do you want to pre-populate the squad (e.g. you create it and manually add members), or do they need to self-join via the invite link?
